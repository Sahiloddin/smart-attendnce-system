# Smart Attendance System — Face Recognition Backend

A production-ready face recognition system for automated attendance using **DeepFace** with the **FaceNet** model. Built as a Flask API that accepts webcam images and returns the recognized student's identity.

---

## 🏗️ Architecture Overview

```
Frontend (React + Webcam)
    │
    ├──▶ POST /api/createdataset/    →  Save face images
    ├──▶ POST /api/retrainmodel/     →  Build/cache face embeddings
    └──▶ POST /api/detectface/       →  Recognize a face
    │
Flask Backend (app.py)
    │
    ├── DeepFace (FaceNet model + MTCNN detector)
    ├── dataset/                     →  Face images organized by person
    └── students.csv                 →  Student metadata (name, roll, email, class)
```

- **Python 3.9 – 3.10** (TensorFlow 2.10 requires Python ≤ 3.10)

## 🚀 How to Run

### Step 1: Create a Virtual Environment

```bash
# Navigate to the FaceRecognition directory
cd FaceRecognition

# Create virtual environment
python -m venv venv

# Activate it
# Windows:
venv\Scripts\activate
# macOS/Linux:
source venv/bin/activate
```

### Step 2: Install Dependencies

```bash
pip install -r requirements.txt
```

**If you face TensorFlow installation issues:**
```bash
# Try installing TensorFlow separately first
pip install tensorflow==2.10.0

# Then install the rest
pip install -r requirements.txt
```

### Step 3: Run the Server

```bash
python app.py
```

The server will start at **http://127.0.0.1:8000**

On first run, it will:
1. Create the `dataset/` directory
2. Create `students.csv`
3. Download and cache the FaceNet model (~92MB, one-time download)

## 📡 API Endpoints

### 1. `POST /api/createdataset/`

Save a webcam capture to the dataset for a student.

**Request Body (JSON):**
```json
{
  "name": "John Doe",
  "roll_number": "101",
  "email": "john@example.com",
  "classroom_name": "Final Year",
  "image": "data:image/jpeg;base64,/9j/4AAQ..."
}
```

**Response (200):**
```json
{
  "message": "Image saved for John Doe",
  "total_images": 5
}
```

### 2. `POST /api/retrainmodel/`

Build face embeddings for all students in a classroom.

**Request Body (JSON):**
```json
{
  "classroom_name": "Final Year"
}
```

**Response (200):**
```json
{
  "message": "Model ready for Final Year",
  "students_count": 3
}
```

### 3. `POST /api/detectface/`

Recognize a face from a webcam frame.

**Request Body (JSON):**
```json
{
  "image": "data:image/jpeg;base64,/9j/4AAQ..."
}
```

**Response (200) — Recognized:**
```json
{
  "name": "John Doe",
  "roll_number": "101",
  "email": "john@example.com",
  "distance": 0.32,
  "confidence": 68.0
}
```

**Response (200) — Unknown:**
```json
{
  "name": "unknown",
  "roll_number": "",
  "email": "",
  "distance": 0.65,
  "confidence": 35.0
}
```

**Response (200) — No face:**
```json
{
  "name": "No face detected",
  "roll_number": "",
  "email": ""
}
```

### 4. `GET /api/health`

Health check endpoint.

## 🔧 Fine-tuning & Dataset Preparation

The `finetune.py` script helps improve recognition accuracy by augmenting your dataset.

### View Dataset Statistics
```bash
python finetune.py
```

### Run Data Augmentation
Generates 6 extra versions of each image (flip, brightness, contrast, rotation):
```bash
python finetune.py --augment
```
This turns 5 images into ~35 images per person!

### Run Face Alignment
Crops and aligns faces to remove background noise:
```bash
python finetune.py --align
```

### Pre-compute Embeddings
Builds the embedding cache for faster first detection:
```bash
python finetune.py --embed
```

### Run Everything
```bash
python finetune.py --all
```

## 🎯 How the Recognition Works

```
Webcam Frame → MTCNN Face Detection → FaceNet Embedding (128-d vector)
                                              │
                                              ▼
                                    Cosine Distance Comparison
                                    against stored embeddings
                                              │
                              ┌───────────────┼───────────────┐
                              ▼               ▼               ▼
                        distance ≤ 0.50   distance > 0.50   no face
                        → RECOGNIZED      → UNKNOWN         → NO FACE
                        (return name,     (return "unknown") → DETECTED
                         roll, email)
```

### Key Parameters

| Parameter | Value | Description |
|---|---|---|
| `MODEL_NAME` | `Facenet` | FaceNet model for 128-d face embeddings |
| `DETECTOR_BACKEND` | `mtcnn` | MTCNN for robust face detection |
| `DISTANCE_METRIC` | `cosine` | Cosine similarity for comparing embeddings |
| `THRESHOLD` | `0.50` | Max cosine distance to accept a match (default FaceNet: 0.40) |

### Adjusting the Threshold

- **Lower threshold (e.g., 0.35)** → Stricter — fewer false positives but more "unknown" results
- **Higher threshold (e.g., 0.60)** → More lenient — recognizes more but may misidentify
- **Current: 0.50** → Good balance for 5-15 images per person

Edit the `THRESHOLD` variable in `app.py` to adjust.

## 💡 Tips for Better Accuracy

### 📸 Image Capture
1. **Capture 15-20 images** per person (minimum 5, but more is better)
2. **Vary the angle**: straight, slight left, slight right, looking up/down
3. **Vary the lighting**: try both well-lit and dim environments
4. **Remove accessories**: glasses, hats, masks during enrollment
5. **Center the face**: face should fill ~60% of the frame
6. **Plain background**: reduces noise in training data

### 🔧 System Setup
7. **Run augmentation** after capturing images: `python finetune.py --augment`
8. **Run alignment** for cleaner training data: `python finetune.py --align`
9. **Good webcam quality**: 720p or higher resolution
10. **Consistent lighting** between enrollment and attendance

### ⚡ Performance
11. **Pre-compute embeddings**: Run `python finetune.py --embed` before attendance
12. **The model is loaded once** at server startup — no reloading per request
13. **Embedding cache** (.pkl files) speeds up subsequent detection calls

## 🔄 Complete Workflow

```
1. Start the Flask server:        python app.py
2. Start the Node.js backend:     cd Backend && node server.js
3. Start the React frontend:      cd Frontend && npm run dev

4. In the browser:
   a. Create a classroom
   b. Go to classroom details
   c. Add students: enter name, roll number, email → capture 15+ photos
   d. Click "Train Model"
   e. Go to "Take Attendance" → turn on camera → faces are auto-detected

5. (Optional) Improve accuracy:
   cd FaceRecognition
   python finetune.py --all
   # Then retrain the model from the UI
```

## 🐛 Troubleshooting

| Problem | Solution |
|---|---|
| `ModuleNotFoundError: No module named 'tensorflow'` | `pip install tensorflow==2.10.0` |
| `No face detected` on every frame | Check webcam permissions, ensure good lighting |
| Everyone shows as `unknown` | Threshold too strict — increase `THRESHOLD` in `app.py` to 0.55-0.60 |
| Wrong person recognized | Need more training images (15+) or run augmentation |
| Server slow on first request | Run `python finetune.py --embed` to pre-compute embeddings |
| `CUDA out of memory` | The system defaults to CPU which is fine for this use case |
| `mtcnn` install fails | `pip install mtcnn==1.0.0` separately |
